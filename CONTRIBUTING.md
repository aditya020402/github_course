## Ways to Contribute

### üêû Bug Reports

If you find a bug, please help us by reporting it.

When reporting a bug, include:
- A clear and descriptive title
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Screenshots, logs, or error messages (if applicable)
- Environment details (OS, browser, language version, etc.)

Please check existing issues before creating a new one to avoid duplicates.

---

### üõ† Bug Fixes

Bug fixes are always welcome!

Steps to contribute a fix:
1. Fork the repository
2. Create a new branch (`fix/short-description`)
3. Make your changes
4. Add or update tests if applicable
5. Ensure the code builds and passes tests
6. Submit a pull request with a clear description

---

### üìö Documentation Improvements

Improving documentation is just as valuable as writing code.

You can help by:
- Fixing typos or grammatical errors
- Improving clarity and examples
- Adding missing information
- Updating outdated documentation

Documentation changes can be submitted directly via a pull request.

### ‚ú® Enhancements and New Features

We welcome ideas for enhancements and new features.

To propose an enhancement:
- Open an issue describing the idea
- Explain the problem it solves
- Provide examples or use cases
- Suggest an implementation approach if possible

Discussion before implementation is encouraged to ensure alignment with
project goals.

---

### üí° Ideas and Suggestions

Have an idea but not sure how to implement it? No problem!

You can:
- Open an issue labeled as an idea or proposal
- Start a discussion with maintainers
- Share design suggestions or improvements

All constructive ideas are welcome.

---

## Pull Request Guidelines

- Keep pull requests focused and small when possible
- Write clear commit messages
- Reference related issues in your PR description
- Be respectful and open to feedback
- Follow the existing code style and conventions

---

## Need Help?

If you have questions or need guidance:
- Check existing issues and discussions
- Reach out to the maintainers
- Ask for clarification before starting major work
